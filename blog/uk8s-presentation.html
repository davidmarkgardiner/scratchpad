üìã Migration Overview

Current: Dedicated AKS

  • Full cluster control
  • Complete administrative access
  • Dedicated infrastructure
  • Full responsibility for maintenance
  • Higher infrastructure costs

New: Shared AKS Platform

  • Namespace-level admin access
  • Shared, managed infrastructure
  • Built-in security policies
  • Cost-effective resource sharing
  • Platform team handles cluster management
Key Principle: You get the same Kubernetes experience, but within a secure, managed, multi-tenant environment.

üîê Security & Access Model

Namespace Isolation

  • Admin access only to your onboarded namespaces
  • Zero access to other tenants' namespaces
  • Network policies prevent cross-namespace ingress
  • Complete isolation from other workloads

Security Policies in Place

Read-only root filesystem No node access Security profile restrictions Network segmentation

Required Roles for Access:

  • Observer Role: Read-only access to resources
  • Admin Role: Full management within your namespace

See platform documentation for detailed role assignments
